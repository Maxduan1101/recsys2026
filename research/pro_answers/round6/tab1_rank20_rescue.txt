优先级：

Seed track → track cf-bpr nearest neighbors
最安全，因为它不是“从文本瞎语义匹配”，而是从对话中已经出现、可能被用户接受的 track 出发扩召相似曲目。适合 “more like this / similar vibe / playlist continuation”。

metadata-qwen3 / attributes-qwen3 seed similarity
用作 guard，不作为主力。它能防止 CF 漂到不相关曲风。

1. 预处理

对所有可用 embedding 做 L2 normalize：

缺失向量直接 mask 掉。不要混 raw cosine；不同模态尺度不同，最终用 rank-normalized score。

2. 构造 seed profile

从历史对话里取 seed tracks：

positive seeds:
- 明确被用户喜欢 / 要求类似 / 上一轮 MOVES_TOWARD_GOAL 的 track
- 如果 goal_progress 对齐还没完全验证，则只给普通历史 track 低权重

negative seeds:
- 用户说不喜欢、不对、换一个
- DOES_NOT_MOVE_TOWARD_GOAL 的 track，前提是你已确认 label 对齐

权重：

Python
运行

    "explicit_like": 1.0,
    "moves_toward_goal": 0.8,
    "previous_music_uncertain": 0.3,
}

w_neg = {
    "explicit_reject": 1.0,
    "does_not_move_toward_goal": 0.7,
}

time_decay = 0.85 ** turns_ago

每个 channel 的 seed profile：

Python
运行
q_c = normalize(
    sum(w_j * decay_j * E_c[j] for j in positive_seeds)
    - 0.6 * sum(w_j * decay_j * E_c[j] for j in negative_seeds)
)

其中 c ∈ {cf, metadata, attributes}。

4. Exact gating rules

一个候选可以替换当前 rank20，当且仅当全部满足：

rank_bm25(candidate) <= 60
rank_ltr(candidate)  <= 40
rank_rrf(candidate)  <= 40
candidate not in base top20
candidate not previously recommended in the same session
candidate artist not already appears >= 3 times in base top20

再加一个 consensus gate：

candidate appears in top50 of at least 3 of 4 component models
OR
candidate appears in top30 of BM25_anchor and top40 of LTR_ensemble

再加一个 margin gate：

rescue_score(candidate) >= rescue_score(base_rank20) + 0.015

如果用 normalized model score 而不是 rank score，可以改成：

z_ltr(candidate) - z_ltr(base_rank20) >= 0.20
AND
z_bm25(candidate) >= z_bm25(base_rank20) - 0.10

意思是：LTR 明显更喜欢它，但 BM25 不能明显反对它。
