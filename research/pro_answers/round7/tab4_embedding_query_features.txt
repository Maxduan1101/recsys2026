这个可能是最高 upside 的专用通道，因为 cover-art 描述不是 BM25/CF 擅长的东西。只要 intent classifier 准，它几乎不会伤 exact-title。

如果 user-CF 在 OOF 里只对 broad/personalized 有用，就保留 interaction；如果整体噪声大，直接删。

9. Hard safety gates

最终 inference 加硬规则：

1. exact title query:
   preserve exact title BM25/LTR candidate in top 1–3
   dense-only candidates forbidden

2. exact artist / album query:
   dense-only candidates forbidden unless they satisfy artist/album constraint

3. cover-art query:
   image dense candidates allowed into full pool
   but require image_z > 1.0 or image_rank <= 50

4. lyrics/theme query:
   lyrics dense candidates allowed
   but require lyrics_rank <= 100

5. broad mood / playlist:
   metadata/attributes dense candidates allowed
   max dense-only candidates in final top20 initially <= 3

6. any candidate with explicit negative feedback:
   remove or strong penalty

7. already recommended track:
   remove unless user explicitly asks to repeat
最推荐的实现目标

不要做“embedding final ranker”。做这个：

Intent-gated Query Dense Candidate + LTR Feature Layer

具体提交前的最小版本：

1. Qwen3 query → metadata-qwen3 top100
2. Qwen3 query → attributes-qwen3 top100
3. lyrics-qwen3 only for lyrics/theme
4. image-siglip2 only for cover-art
5. user-cf only as feature
6. exact-title / exact-artist cases forbid dense-only candidates
7. train honest OOF LTR with dense hard negatives

判断是否值得保留的唯一标准：

OOF overall nDCG@20 上升
AND exact-title subset 不下降
AND dense channels 的 gain 来自 gated categories，而不是随机扰动 BM25 头部

一句话：下一个最有希望的 embedding 路线不是“再找相似歌”，而是“把当前用户请求编码成多模态 query，用 dense retrieval 扩召难被 BM25 命中的语义/歌词/封面候选，再交给 LTR 学会什么时候信它”。
