优先级：

Seed track → track cf-bpr nearest neighbors
最安全，因为它不是“从文本瞎语义匹配”，而是从对话中已经出现、可能被用户接受的 track 出发扩召相似曲目。适合 “more like this / similar vibe / playlist continuation”。

用作 guard，不作为主力。它能防止 CF 漂到不相关曲风。

No-label 决策检查

在提交前，对四个包跑同一套 Blind-A-like 80-row panel + full dev response audit。

1. Ranking invariance check

必须全部通过：

top20 track IDs 完全一致
nDCG/candidate ranking 文件完全一致
catalog diversity 完全一致或只因格式统计差异微小变化

如果 response package 意外改了排序，直接拒绝。

2. Grounding check

硬门槛：

invalid title/artist/album mention = 0
hallucinated genre/mood/instrument = 0
unclean display artifacts = 0
blacklisted noisy tags = 0

重点 ban：

sleep
to live by
cds I own
seen live
favorites
misc user-library tags
raw bracket/list artifacts
unknown album if album missing

任何包出现 hallucinated metadata，直接降级。

3. Mechanicalness check

统计 opening、句式、模板复用。

硬阈值：

top opening 5-gram share <= 10% full dev
top opening 5-gram count <= 8 / 80 Blind-like panel
exact sentence reuse <= 5% full dev
same template family <= 25% full dev
same template family <= 30% Blind-like panel
generic explanation phrase rate <= 8%

generic explanation phrase 包括：

"matches your taste"
"fits your preferences"
"good choice for you"
"based on your profile"
"has the vibe you want"

如果没有具体 track/artist/tag/history 支撑，这些都算泛化废话。

决策：

compact_broad 如果 mechanicalness 任一项超阈值，直接拒绝。
compact_clean 如果比 judge_clean_mix 高出 >2 percentage points 的 mechanical flags，不作为下一次提交。
4. Personalization coverage check

不是每行都强行 personalization，而是按 query 类型。

对 broad / mood / playlist / similar rows：

>= 70% responses mention one grounded context cue

grounded context cue 可以是：

previous liked track
current stated mood
requested genre
requested artist
conversation goal
profile preference
negative feedback constraint

对 exact title / exact artist / exact album rows：

unnecessary profile/history mention <= 15%

因为 exact query 乱 personalization 会显得不听话。

硬规则：

history/profile mention 必须能在 conversation/profile 中找到证据
不能说 "you liked X" 除非 X 明确出现过 positive signal
5. Naturalness check

用规则近似 Gemini 风险。

推荐阈值：

mean response length: 35–75 words
rows < 20 words: <= 5%
rows > 95 words: <= 5%
one-sentence responses: <= 35%
responses with 4+ comma-separated adjectives: <= 10%
responses with list-like metadata dumping: <= 5%

自然解释应该像：

I’d start with {track} by {artist}; it keeps the {grounded mood/genre} direction from your last request while avoiding {negative constraint if any}.

不应该像：

This recommendation matches your vibe, energy, mood, taste, and preferences with strong genre alignment.
6. Lexical floor

不要为了自然度牺牲太多 lexical。

下一次提交候选必须满足：

dev lexical >= 0.190
Blind-A-like Distinct-2 >= 0.58

因此：

judge_clean_mix passes
compact_clean passes
compact_broad passes but mechanical-risk gated
judge_balanced_mix fails lexical floor unless it clearly wins naturalness audit

judge_balanced_mix 的 Blind-A Distinct-2=0.55719，不是灾难，但低于我会设的 safe floor。它只能作为“Gemini 风格明显更强”的替代，不是默认提交。

Pairwise no-label audit

80 Blind-A-like rows

    exact query
