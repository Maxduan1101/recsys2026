主人，完整结论：现在最安全的 embedding 用法不是 seed/user 继续加权，也不是新增 dense retrieval，而是做“BM25/LTR 候选内部的 embedding-consistency features”。也就是：candidate pool 完全不变，top-rank 不做 postprocess，embedding 只告诉 LightGBM：这个候选是否与 BM25 头部、当前候选簇、明确正负反馈、用户 CF 方向一致。

官方 Music-CRS 任务要求系统根据 dialogue history 和 user context 产出 ranked tracks 与 response text；官方也提供 multimodal track/user embeddings，适合做这种上下文排序证据，而不是盲目替代 BM25。

1. 最安全优先级

按风险从低到高：

P0: BM25-head anchor agreement features
P0: candidate-cluster consistency features
P1: exact/generic intent-gated user-CF affinity features
P1: explicit feedback-only seed margin features
P2: query-to-track dense scores as features, not retrieval
P3: gated dense candidate evidence, only for cover-art / lyrics / broad mood
P4: dense retrieval / seed-CF postprocess，暂时不要做

既然 seed-CF tail rescue neutral，而且 seed/user embedding LTR features 也可能 hurt，下一步不要继续强化 seed/user。优先做不依赖 noisy feedback alignment 的：

BM25 head → embedding centroid → candidate agreement

这类特征最稳，因为它只在当前 BM25 已经找出的候选之间重排，不新增候选、不硬替换 top20。

2. 全局预处理

对每个 channel 的 track embedding 做 L2 normalization：

Python
运行
E_c[i] = emb_c[i] / (np.linalg.norm(emb_c[i]) + 1e-12)

channels:

Python
运行

  "meta": metadata_qwen3,
  "attr": attributes_qwen3,
  "lyrics": lyrics_qwen3,
  "audio": audio_laion_clap,
  "image": image_siglip2,
  "track_cf": track_cf_bpr,
}

user CF:

Python
运行
U[u] = user_cf_bpr[u] / (np.linalg.norm(user_cf_bpr[u]) + 1e-12)

每个 candidate group 是一个 session-turn：

Python
运行
G = candidates for one session-turn

group 内 robust z-score：

Python
运行
def robust_z(xs):
    med = median(xs_valid)
    iqr = percentile(xs_valid, 75) - percentile(xs_valid, 25)
    if iqr < 1e-6:
        return 0.0
    return clip((x - med) / (iqr + 1e-6), -5.0, 5.0)

rank score：

Python
运行
rank_score(r) = 1.0 / log2(r + 1)
3. P0 特征一：BM25-head anchor agreement

这是最推荐先做的 block。

3.1 构造 BM25 anchor centroid

对每个 query/group，取当前 BM25-derived rank 的 top-k：

Python
运行
H_k(q) = top k BM25 candidates
k = 5 or 10

建议先用：

k = 5 for exact / narrow query
k = 10 for broad / playlist / mood query

过滤 anchor：

Python
运行
valid_anchor(j, c) =
    has_embedding(c, j)
    and not explicitly_rejected(j)
    and not duplicate_removed(j)

anchor 权重：

Python
运行
w_j = 1.0 / log2(bm25_rank_j + 1)

channel centroid：

Python
运行
A_c(q) = normalize(
    sum_{j in H_k(q), valid_anchor(j,c)} w_j * E_c[j]
)

如果有效 anchor 少于 2 个：

Python
运行
A_c(q) = missing

6. P1 特征四：explicit feedback-only seed margin

既然 seed-CF rescue neutral，seed features 只能非常保守地用，而且只用明确 feedback。

不要用所有历史推荐当 positive seed。只用：

positive seed:
- user explicitly liked
- user asked "more like this"
- accepted / moves-toward-goal only if label alignment 已验证

negative seed:
- user explicitly rejected
- user said wrong / not this / too slow / too pop / not that artist

seed centroid：

Python
运行
Pos_c(q) = normalize(sum_s w_s * E_c[s])
Neg_c(q) = normalize(sum_s w_s * E_c[s])

权重：

Python
运行
w_s = feedback_strength_s * 0.85 ** turns_ago_s

candidate features：

Python
运行
seed_pos_cos_c[i] = dot(E_c[i], Pos_c(q))
seed_neg_cos_c[i] = dot(E_c[i], Neg_c(q))

seed_margin_c[i] = seed_pos_cos_c[i] - 0.7 * seed_neg_cos_c[i]

12. Ablation protocol

Use same folds, same candidate pool, same labels, same LightGBM params.

A0 baseline
current BM25-derived candidates + current LightGBM features
A1 anchor-only

Add only:

BM25-head anchor agreement:
metadata, attributes, track_cf

Accept if:

OOF nDCG@20 >= baseline + 0.0005
exact-title subset delta >= -0.0002
top5 nDCG delta >= -0.0002
changed top5 rows not excessive

This is the best first candidate.

A2 cluster-only

Add candidate-cluster consistency.

Accept if:

broad / playlist / mood improves
exact-title unchanged
overall OOF non-negative

Reject if exact-title drops; cluster features often punish singleton exact matches.

A3 anchor + cluster

Accept only if better than A1.

If A1 improves but A3 flat/hurts, keep A1.

A4 user-CF gated only

Add user-CF features with exact-title disabled.

Accept if:

personalized/broad subset improves
overall OOF >= A1 - 0.0002
exact-title unchanged

Reject if feature importance shows user_cf_z globally dominates. That means the model may overfit user prior.

A5 explicit seed margin only

Add seed margin from explicit feedback only.

Accept if:

similar-to-this / feedback-heavy subset improves
overall OOF non-negative
negative feedback cases improve or unchanged

Reject if seed features help only train folds but not OOF.

A6 query dense evidence only

Candidate pool unchanged. Add dense hit/rank features for existing BM25 candidates.

Accept if:

OOF nDCG@20 improves
exact-title unchanged
lyrics/theme improves when lyrics enabled
cover-art improves when image enabled
A7 gated candidate addition

Only after A1–A6 show safe gains.

Rules:

Do not add dense candidates for exact-title / exact-artist / exact-album.
For broad/mood: add max 20 dense candidates to candidate pool, not direct final top20.
For lyrics/theme: add lyrics top50.
For cover-art: add image top100.

Accept only if:

gold_in_pool@300 increases
OOF nDCG@20 increases
dense-only false positives do not flood feature importance
exact-title unchanged

14. Rollback signals

Rollback immediately if any of these happen:

1. OOF nDCG@20 drops by more than 0.0003–0.0005.

2. exact-title subset drops, even if overall rises slightly.

3. top5 nDCG drops.

4. BM25 top1/top3 gold demotion count increases materially.

5. Blind-A-shaped 80-row panel p10 delta is negative enough, e.g. < -0.001.

6. LightGBM feature importance is dominated by raw user_cf or seed_cf features.

7. Gains are isolated to one tiny subset with unstable fold variance.

8. Dense/evidence features increase changed top5 rows too much.

9. Candidate pool recall rises but final nDCG does not.

10. Embedding features promote already-recommended or explicitly rejected tracks.

For your current setup, I would require:

minimum accept:
OOF nDCG@20 >= baseline + 0.0005
exact-title delta >= 0
top5 nDCG delta >= -0.0002
Blind-A-shaped panel mean > 0
Blind-A-shaped p10 not strongly negative

If OOF gain is only +0.0001 and exact-title/top5 worsens, do not submit.

15. Concrete recommendation

Implement in this order:

1. A1 anchor-only:
   BM25-head centroid agreement over metadata, attributes, track_cf.

2. A1-small:
   same as A1 but exact-title embedding disabled.

3. A1 + cluster:
   add candidate-cluster consistency only if exact-title safe.

4. A1 + gated user-CF:
   user-CF only for broad/personalized/tail interactions.

5. A1 + explicit seed margin:
   only explicit positive/negative feedback.

6. Dense evidence features:
   dense rank/hit features for existing candidates only.

7. Gated candidate addition:
   only for lyrics/theme, cover-art, broad mood, never exact-title.

The best single next experiment is:

BM25-head anchor agreement features only,
with exact-title cases disabled,
candidate pool unchanged,
no postprocess,
no dense-only candidates.

That gives embeddings a realistic path to improve honest OOF nDCG@20 by helping LightGBM decide which BM25 tail candidates are semantically coherent, while minimizing the risk of disturbing the BM25/exact-match head.
