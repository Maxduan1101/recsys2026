结论：第一优先做 seed-CF + user-CF，但只进尾部

优先级：

Seed track → track cf-bpr nearest neighbors
最安全，因为它不是“从文本瞎语义匹配”，而是从对话中已经出现、可能被用户接受的 track 出发扩召相似曲目。适合 “more like this / similar vibe / playlist continuation”。

User cf-bpr → track cf-bpr score
作为个性化 prior 很有用，但风险比 seed-CF 大，因为它可能推荐“用户长期偏好”而不是“当前 query 目标”。所以第一版只做 feature 或 tail gate，不要直接冲 top-10。

metadata-qwen3 / attributes-qwen3 seed similarity
用作 guard，不作为主力。它能防止 CF 漂到不相关曲风。
lyrics/audio/image 暂时不要全局启用；只在后续按 intent gate 加。

Exact scoring
1. 预处理

对所有可用 embedding 做 L2 normalize：

Python
运行
E_cf[i]      = normalize(track_cf_bpr[i])
E_meta[i]    = normalize(track_metadata_qwen3[i])
E_attr[i]    = normalize(track_attributes_qwen3[i])
U_cf[u]      = normalize(user_cf_bpr[u])

缺失向量直接 mask 掉。不要混 raw cosine；不同模态尺度不同，最终用 rank-normalized score。

2. 构造 seed profile

从历史对话里取 seed tracks：

positive seeds:
- 明确被用户喜欢 / 要求类似 / 上一轮 MOVES_TOWARD_GOAL 的 track
- 如果 goal_progress 对齐还没完全验证，则只给普通历史 track 低权重

negative seeds:
- 用户说不喜欢、不对、换一个
- DOES_NOT_MOVE_TOWARD_GOAL 的 track，前提是你已确认 label 对齐

权重：

Python
运行
w_pos = {
    "explicit_like": 1.0,
    "moves_toward_goal": 0.8,
    "previous_music_uncertain": 0.3,
}

w_neg = {
    "explicit_reject": 1.0,
    "does_not_move_toward_goal": 0.7,
}

time_decay = 0.85 ** turns_ago

每个 channel 的 seed profile：

Python
运行
q_c = normalize(
    sum(w_j * decay_j * E_c[j] for j in positive_seeds)
    - 0.6 * sum(w_j * decay_j * E_c[j] for j in negative_seeds)
)

其中 c ∈ {cf, metadata, attributes}。

3. Embedding candidate scores

对每个 channel 检索 top-K：

Python
运行
sim_seed_cf[i]   = dot(q_cf,   E_cf[i])
sim_seed_meta[i] = dot(q_meta, E_meta[i])
sim_seed_attr[i] = dot(q_attr, E_attr[i])

sim_user_cf[i] = dot(U_cf[user_id], E_cf[i])

建议 K：

Python
运行
K_seed_cf = 100
K_seed_meta = 50
K_seed_attr = 50
K_user_cf = 200

把 rank 转成稳健分数：

Python
运行
R(rank) = 1 / log2(rank + 1)

然后：

Python
运行
S_seed(i) =
    0.50 * R(rank_seed_cf(i))
  + 0.30 * R(rank_seed_meta(i))
  + 0.20 * R(rank_seed_attr(i))

S_user(i) = R(rank_user_cf(i))

如果某 item 不在某 channel top-K，那个 channel 给 0。

初始权重故意小：embedding/CF 只能帮助 BM25 tail 排序，不能推翻 BM25 anchor。

A1. 只重排 BM25 16–20，不引入新候选
