Best practical design

The strongest practical design is a late-fusion, multi-channel retriever/reranker, not a single concatenated embedding. Keep BM25 as the lexical anchor, then add independent candidate channels for Qwen text embeddings, seed-track similarity, user–track CF, and modality-specific expansion. This fits the official setup: Music-CRS is multi-turn conversational recommendation with pre-extracted multimodal track/user embeddings, and the evaluator ranks up to 20 track IDs per turn using nDCG@1/10/20 plus diversity metrics. 
NLP4MUSA
+1

The core rule is: retrieve broadly, calibrate each channel separately, then rerank with gated features. Do not average raw scores across modalities, and do not hard-filter the catalog. The official baseline warns that inference should retrieve from all_tracks, not a restricted subset.

    norms = np.linalg.norm(X, axis=1)
    valid = norms > eps
    Xn = np.zeros_like(X, dtype=np.float32)

3. Seed similarity from conversation history

This challenge is especially suited to seed expansion because the conversation history contains previous music role messages whose content is a track UUID, and the evaluator’s ground truth parser takes the current turn’s second conversation item as the target track ID.

+1

Maintain seed sets per session:

positive_seeds:
  previous recommended track IDs that the user liked, accepted, refined from, or asked for "more like this"

neutral_seeds:
  previous recommendations without clear feedback

negative_seeds:
  previous recommendations followed by "not that", "too heavy", "less pop", "wrong era", etc.

Use recency-weighted seed centroids:

Python
运行
w_t = feedback_weight * gamma ** (current_turn - seed_turn)
gamma = 0.75  # start value

pos_centroid_m = normalize(sum(w_t * E_norm[m][track_id] for positive/neutral seeds))
neg_centroid_m = normalize(sum(w_t * E_norm[m][track_id] for negative seeds))

For each candidate i and modality m:

Python
运行
seed_sim_m(i) = max(
    dot(E_norm[m][i], pos_centroid_m),
    max_seed_dot(E_norm[m][i], E_norm[m][seed_ids])
)

neg_sim_m(i) = max_negative_seed_dot(...)

Use both centroid and max-seed similarity. The centroid captures the general taste direction; max-seed similarity prevents mixed tastes from averaging into nonsense.

Good initial seed channels:

metadata-qwen3        general semantic similarity
attributes-qwen3      mood/style/instrument similarity
audio-laion_clap      sonic similarity
cf-bpr track          co-listening similarity
image-siglip2         cover-art / album visual similarity
lyrics-qwen3          lyric-topic similarity

Do not treat every previous recommendation as positive. If the next user message says “not quite,” “less heavy,” “I wanted something older,” or “that’s not it,” move that previous track to the negative set for the relevant modality.

6. Final top-20 selection

After reranking, apply a light MMR pass:

Python
运行
final = []
while len(final) < 20:
    pick item maximizing:
        rank_score(item)
      - lambda_artist * same_artist_count(item, final)
      - lambda_album  * same_album_count(item, final)
      - lambda_dense  * max_similarity_to_final(item)

Use small penalties. The primary metric is still nDCG with a single ground-truth item per turn, so too much diversity can hurt. A reasonable start:

Python
运行
lambda_artist = 0.03
lambda_album  = 0.04
lambda_dense  = 0.02

For “find one specific song” intents, disable most diversity penalties. For “give me several different artists” intents, increase same-artist penalty.

The output must contain unique predicted_track_ids, ordered by relevance, up to 20 IDs per turn.

7. Modality-specific failure modes
Modality	Good for	Failure mode	Mitigation
metadata-qwen3	title, artist, album, tags, release-era text	Overmatches surface metadata; can miss sonic mood	Use with BM25 and attributes; add exact entity features
attributes-qwen3	mood, genre, instrumentation, activity	Generic moods create hubs: “chill,” “energetic,” “sad”	Calibrate by rank/z-score; add seed/audio features
lyrics-qwen3	lyric themes, story, quoted words, language	Bad for instrumental/sonic queries; lyrics may be missing/noisy	Gate only for lyric/theme/vocal queries
audio-laion_clap	sound, timbre, energy, production, “sounds like”	Weak for artist/title/year; audio embeddings can cluster by production artifacts	Use mostly for seed expansion and sonic queries; never dominate exact searches
image-siglip2	cover art, visual clues, album identity	Cover art is not music; same album art can rank all album tracks	Gate only on visual/cover queries or album-art seed cases
cf-bpr track	co-listening similarity, “more like this”	Popularity bias; opaque jumps across genres	Use as a feature, not a hard retriever; combine with content similarity
user cf-bpr	warm-user personalization	Can override current intent; poor for known-item search	Gate by intent and downweight for exact/visual/lyric searches

CLAP-style audio models align language and audio in a shared space, which is useful for zero-shot sonic retrieval, but that also means broad natural-language descriptors may retrieve acoustically plausible tracks rather than catalog-correct tracks.

 SigLIP2 is designed for image-text retrieval and visual representations, so it is useful for cover-art clues but should not be treated as a music-content embedding.

def l2_normalize_with_mask(X, eps=1e-12):
    X = np.asarray(X, dtype=np.float32)
    ok = np.isfinite(X).all(axis=1)
    norm = np.linalg.norm(X, axis=1)
    ok &= norm > eps

    Xn = np.zeros_like(X, dtype=np.float32)
    Xn[ok] = X[ok] / norm[ok, None]
    return Xn, ok

def robust_z(scores, eps=1e-9):
    scores = np.asarray(scores, dtype=np.float32)
    med = np.median(scores)
    mad = np.median(np.abs(scores - med))
    z = (scores - med) / (1.4826 * mad + eps)
    return np.clip(z, -4.0, 4.0)

关键补充：官方给的是 track/user embeddings，不是 query embeddings。若规则和算力允许，可用相同 encoder 给当前 query 生成 Qwen3 / CLAP-text / SigLIP-text query embedding；若只想严格依赖官方预计算 embedding，则 dense direct query 通道先不做，把官方 embedding 用在 seed expansion、user-CF、reranking features 上。用户 embedding 表提供 user_id 和 128-d cf-bpr，并有 train/test_warm/test_cold splits。

4. Seed similarity 设计

官方 conversation 里有 conversations 列，包含 user/music role；music role 的 content 是 track UUID，适合作为历史 seed。数据还包含 8 个 turn 的 goal-progress assessments，可在训练/dev 分析时使用，但 blind inference 不应依赖未来或不可见标签。

4.2 Seed score

每个 modality 单独算：

Python
运行
pos_centroid_m = normalize(sum(w_s * E_m[seed_s] for seed_s in positive_or_neutral))
neg_centroid_m = normalize(sum(w_s * E_m[seed_s] for seed_s in negative))

seed_score_m(i) = max(
    dot(E_m[i], pos_centroid_m),
    max(dot(E_m[i], E_m[s]) for s in positive_or_neutral_seeds)
)

neg_score_m(i) = max(
    dot(E_m[i], neg_centroid_m),
    max(dot(E_m[i], E_m[s]) for s in negative_seeds)
)

保留 centroid 和 max-seed 两种分数。只用 centroid 会把多个风格平均成一个不存在的“中间口味”；只用 max-seed 又容易被单个偶然 seed 带偏。

5. User-CF scoring

cf-bpr 的最稳用法分两种：

user → track:
  raw dot product: dot(user_cf_raw, track_cf_raw)

track → track:
  cosine over L2-normalized track cf-bpr

BPR 是面向 implicit feedback 的 personalized ranking 方法，常用于学习 user/item latent factors 并优化排序目标，因此 user-item CF 分数最自然的初始形式就是用户向量与物品向量的 compatibility score。
arXiv

实现：

Python
运行
def user_cf_scores(user_id, candidate_track_idx):
    if user_id not in user_cf:
        return np.zeros(len(candidate_track_idx)), False

    u = user_cf_raw[user_id]              # [128]
    V = track_cf_raw[candidate_track_idx] # [n, 128]
    s = V @ u
    return robust_z(s), True

def postprocess(ranked_items, topk=20):
    seen = set()
    out = []

    for item in ranked_items:
        if item.track_id in seen:
            continue
        if not item.valid_track_id:
            continue
        out.append(item.track_id)
        seen.add(item.track_id)
        if len(out) == topk:
            break

    return out

8. 各 modality 的失败模式和防护
Modality	适合	典型失败	防护
BM25	exact title/artist/album/year/tag	同名歌、cover、拼写变体、语义 mood 弱	加 metadata-qwen、artist/album exact features
metadata-qwen3	title/artist/album/genre 语义	对 “chill/intense/sad” 这类泛词产生 hub	rank+z 校准，和 attributes/audio 共同投票

lyrics-qwen3	歌词主题、故事、片段、语言	对纯声音需求无效；歌词缺失/噪声	只在 lyric intent 强启用
audio-laion_clap	sonic texture、energy、lo-fi、acoustic、aggressive	不擅长 exact song/artist；可能按 production artifact 聚类	主要用于 seed 和 audio intent，不让它主导 exact query
image-siglip2	cover art、视觉记忆、album identity	封面不是音乐；同 album 多 track 混淆	只在 visual intent 启用，加 album-level rerank
track cf-bpr	co-listening、more-like-this	popularity bias、跨 genre 跳跃难解释	用 cosine 做 item-item，作为 feature 不硬过滤
user cf-bpr	个性化兜底、broad discovery	覆盖当前明确意图	exact/lyrics/visual intent 下大幅降权

Python
运行
def generate_candidates(state, user_id, K):
    C = CandidateSet()

    # 1. lexical anchor
    C.add("bm25", bm25.search(state.retrieval_text, topk=K.bm25))

    # 2. direct dense query, only when query encoders are available
    if encoders.qwen_available:
        q_meta = encoders.qwen_embed(state.metadata_query)
        C.add("q_metadata", faiss["metadata"].search(q_meta, K.metadata))

        q_attr = encoders.qwen_embed(state.attribute_query)
        C.add("q_attributes", faiss["attributes"].search(q_attr, K.attributes))

        if state.intent.lyric:
            q_lyr = encoders.qwen_embed(state.lyric_query)
            C.add("q_lyrics", faiss["lyrics"].search(q_lyr, K.lyrics))

    if state.intent.audio and encoders.clap_text_available:
        q_audio = encoders.clap_text_embed(state.audio_query)
        C.add("q_audio", faiss["audio"].search(q_audio, K.audio))

    if state.intent.visual and encoders.siglip_text_available:
        q_img = encoders.siglip_text_embed(state.visual_query)
        C.add("q_image", faiss["image"].search(q_img, K.image))

    # 3. official-embedding seed expansion
    for ch in ["metadata", "attributes", "track_cf", "audio"]:
        if state.has_positive_seeds(ch):
            C.add(f"seed_{ch}", seed_knn(ch, state, K.seed))

    if state.intent.lyric and state.has_positive_seeds("lyrics"):
        C.add("seed_lyrics", seed_knn("lyrics", state, K.seed))

    if state.intent.visual and state.has_positive_seeds("image"):
        C.add("seed_image", seed_knn("image", state, K.seed))

    # 4. user CF
    if user_id in store.user_cf and not state.intent.exact_strong:
        C.add("user_cf", topk_user_cf(user_id, K.user_cf))

    return C

Python
运行
def rank_candidates(C, state, user_id):
    F = feature_builder.build(C, state, user_id)

    weights = get_base_weights()
    weights = apply_intent_gates(weights, state.intent)

    score = np.zeros(len(F))

    for name, w in weights.items():
        score += w * F[f"{name}_z"]

    score += 0.20 * F["exact_title_match"]
    score += 0.15 * F["exact_artist_match"]
    score += 0.08 * F["year_match"]
    score += 0.04 * F["popularity_z"]

    score -= 0.15 * F["already_recommended"]
    score -= 0.10 * F["neg_seed_max_z"]
    score -= 0.03 * F["same_artist_repeat_count"]

    return sort_by_score(C.track_ids, score)
